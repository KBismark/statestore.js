Getting started

First of all, Statestore as the name may suggest is a simple store for managing states in your applications. It allows you to publicise some parts or all of the state of a particular component or function in order to make data accessible to other parts of your application with ease.

Let's get you started!

Installation

Install statestore from the Node Package Manager. 

npm install statestorejs

If you prefer to use Yarn, 
yarn add statestorejs

A Simple Usage of Statestore

Statestore is purely a JavaScript library and not tied to the usage of a particular framework or library. It can be used to manage states efficiently in any JavaScript environment. In the examples below, we show how Statestore can be used to optimize your React applications.

In this first example, we do not make use of statestore for state management and data access. It's obvious how difficult it is to communicate between components without causing re-render of our entire application. It becomes more difficult and expensive when some state needs to be passed or shared accross components with different immediate parent components. In that case, state needs to be raised higher in the component tree to some great grand parent component and re-render downwards to the last child component every time state changes.

This is the optimized version of the SimpleCircuit app above using statestore. In this version, we use statestore to handle states that needs to be accessed in other parts of our application. Even though both examples result in the same output, the example with statestore is more performant and efficient.
Notable among the reason is the fact that both the SimpleCircuit and the SwitchButton components will never re-render in the application since state does not affect them in any way. However, same components in the first example re-render every time state changes despite state having no effect on them.
Another optimization is applied to FirstRoomOnStatus component. It only re-renders when count in the state storage increases, which is exactly what it is supposed to do. However, whether count was increased or not, FirstRoomOnStatus had to re-render every time state changes in the first example.

Statestore shines even without the use of optimizations provided in the React library like the memoization, useMemo or the likes. 


import React from 'react'
import { configureForReact, useStateStore, createStore, updateStore, getStore } from 'statestorejs'
configureForReact(React); // Configuration for using with react or react-natve

export default function SimpleCircuit(){
    createStore('storageName', 'storeId', {status: false, count: 0})
    return (
        <div>
            <Room id={1} />
            <Room id={2} />
            <SwitchButton />
            <FirstRoomOnStatus />
        </div>
    )
}

function SwitchButton(){
    const toggleSwitch = ()=>{
        const { status, count } = getStore('storageName', 'storeId')
        const reactors = !status?['status','count']: ['status'];
        updateStore(
            'storageName', 'storeId', 
            { actors: reactors, store:{ status, count:!status?count+1:count } }
        )
    }
    return <button onClick={toggleSwitch} > Switch </button>
}
